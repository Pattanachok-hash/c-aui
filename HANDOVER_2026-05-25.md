## Things To Watch Carefully

- Do not commit secrets or local tooling:
  - `backend/.env`
  - `.claude/`
  - local design zip/folders
  - logs
  - `__pycache__`
- In `transport-dashboard`, `.claude/settings.local.json` is modified locally. Do not commit it.
- In `warehouse-scanner`, these are untracked and should not be committed unless explicitly approved:
  - `.claude/`
  - `AGENTS.md`
  - `warehouse-scanner-redesign/`
- `api.c-aui.com` must serve the backend API, not the portal frontend.
  - If it shows the SMART EXPORT portal page, Railway is deploying the wrong thing.
  - Check Railway builder/source/root directory/Dockerfile setup.
- Railway backend uses the `PORT` env provided by Railway.
  - User has had past issues where port `8080` was needed.
  - If Railway says "Application failed to respond", check deployment logs and public networking port.
- SSO tokens are passed in URL fragments.
  - Fragments are not sent to the server.
  - Receiver clears the hash immediately.
  - Keep all production SSO on HTTPS only.
- GitHub Pages and Railway deploys can lag.
  - Hard refresh browser after deploy.
  - Check latest commit hash in deployment if behavior looks stale.
- Supabase JWT is ES256/asymmetric.
  - Portal backend verifies by calling Supabase `/auth/v1/user`.
  - Warehouse/Transport hardening verifies against Supabase/JWKS style behavior in their own code.
